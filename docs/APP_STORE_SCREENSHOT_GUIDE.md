## 這六張在說一個故事

像 Apple 發表會，**一張只講一件事**。使用者滑過去，應該感受到：

> 有人幫你想好了。你不用完美。照常過就好。

**不要出現：** 百分比儀表板、KPI 大字、箭頭飆升、Before/After、「神器」「暴瘦」「AI 引擎」  
**要出現：** 留白、一句話、真實生活感、茶棕 calm 色（`#F4F2EE` / `#7A756D`）

### 敘事順序（轉換率最佳實踐）

```
01 情感入口 → 02 差異化能力 → 03 品牌性格 → 04 可執行性 → 05 安靜信任 → 06 收束認同
```

---

## 全系列視覺規範

| 項目 | 規範 |
|------|------|
| 外框背景 | `#F4F2EE` 或極淡 gradient，無 busy pattern |
| 標題字級 | 大、少字（6–12 字為佳） |
| 副標 | 一行，14–20 字 |
| 手機 mockup | 可選；若用，留大量留白，不要塞滿 |
| 狀態列 | 統一 9:41，滿格 Wi‑Fi，勿顯示低電量 |
| 測試帳號 | 暱稱用「阿凱」或「小安」，不用 `test@test.com` 露出 |
| 禁止畫面 | 「會員準備中」、Apple Health 連接、空白 onboarding、錯誤 toast |

# 01 · hero

**檔名：** `01-hero.png`

## 1. 使用頁面

**Today** — `/dashboard`

## 2. 最適合的資料狀態

| 項目 | 建議 |
|------|------|
| 時間 | 週間中午或傍晚（11:30–13:00 或 17:30–19:00） |
| 餐次 | **午餐**或**晚餐** tab 已選，顯示「這餐還沒記錄」或剛骰出一組外食 |
| 今日紀錄 | 已記 1 餐（早餐或前一餐），**不要**已達 100% 滿格 |
| 熱量列 | 頂部 intake 有數字，但**不要**強調赤字／超標紅色 |
| 骰子預覽 | 顯示一組**熟悉台灣外食**（例：梁社漢便當、7-ELEVEN 組合），非 placeholder |
| 再健 | 可見陪伴短句，非 error 狀態 |

**情緒：** 一天還在進行中，還來得及 — 不是「你又失败了」。

## 3. 標題文案

```
今天吃什麼？
```

## 4. 副標題

```
不用自己想。照著做就好。
```

## 5. 是否需要真實食物照片

**否。** 以 UI 骰子／菜單卡片為主。若卡片有食物圖，用 App 內建品牌圖即可，勿另外 P 假圖。

# 02 · photo-ai

**檔名：** `02-photo-ai.png`

## 1. 使用頁面

**Photo** — `/dashboard` → 拍照記錄流程（`PhotoLogSheet` 或剛加入今日列表）

## 2. 最適合的資料狀態

| 項目 | 建議 |
|------|------|
| 流程階段 | **二選一：** (A) 相機／相簿選擇後、AI 已估算完成；或 (B) 食物已加入列表，顯示 `learning` 後的完整 kcal |
| 食物 | **家常、辨識度高**：便當、滷肉飯、雞腿便當 — 非 exotic 或 messy 擺盤 |
| 表單 | **零表單** — 畫面中不可見長表單、手動輸入欄位 |
| kcal 顯示 | 有數字但**小字、附在紀錄旁**，不要整屏「700 kcal!!!」 |
| 狀態 | 成功加入，無 error toast |

**情緒：** 拍一下，繼續生活 — 不是「精准识别引擎 v2」。

## 3. 標題文案

```
拍一下就好。
```

## 4. 副標題

```
不用算熱量。我們幫你記。
```

## 5. 是否需要真實食物照片

**是 — 必須。**  
用**真實拍攝**的家常便當／外食（可自己準備後拍）。  
光線自然、俯角 45°、盤面乾淨。  
**不要** stock photo 感、不要 filter 過重、不要減肥餐草席擺拍。

# 03 · recovery

**檔名：** `03-recovery.png`

## 1. 使用頁面

**Today（Recovery / 熱量銀行）** — `/dashboard`

## 2. 最適合的資料狀態

| 項目 | 建議 |
|------|------|
| 情境 | 昨日或今日**稍微多吃** — 銀行顯示 gentle recovery 文案，**不是**紅色警告 |
| 文案 | 顯示 Oura 式陪伴句（如「明天慢慢走回來」），非「超標 +800 kcal」 |
| 進度 | intake 略高於 target 即可，**不要**誇張爆表 |
| 色調 | 茶棕、sage 綠完成態 — **禁止** error 紅大面积 |
| 其他 UI | 可見今日餐次列表 2–3 筆，生活感 |

**情緒：** 吃多了沒關係 — 這是 BetterBit 最不像減肥 App 的一張。

## 3. 標題文案

```
吃多了，也沒關係。
```

## 4. 副標題

```
明天再慢慢走回來。
```

## 5. 是否需要真實食物照片

**否。** 重點是**文案與 calm UI**，不是食物本身。列表內小 icon 即可。

# 04 · week

**檔名：** `04-week.png`

## 1. 使用頁面

**Week** — `/weekly`

## 2. 最適合的資料狀態

| 項目 | 建議 |
|------|------|
| 週次 | 本週計畫已生成，`week_number ≥ 2` 更有「持續感」 |
| 內容 | 三餐 + 運動混合：**外食 + 便利店 + 自煮** 各至少一項 |
| 完成度 | 週內 2–3 天有打卡痕跡，**不要**全灰或全綠 |
| 運動 | 顯示 1–2 個 mainstream 項目（快走、深蹲），非 hardcore 健身 |
| 捲動位置 | 週概覽 + 今日可見區，**不要**只露底部 footer |

**情緒：** 一週已排好 — 像翻開一本安靜的週記，不是 Gantt chart。

## 3. 標題文案

```
這週，都幫你排好了。
```

## 4. 副標題

```
吃、動、休息。照常過。
```

## 5. 是否需要真實食物照片

**否。** 以週計畫文字／卡片為主。若有 thumbnail，用 App 內建即可。

# 05 · progress

**檔名：** `05-progress.png`

## 1. 使用頁面

**Progress** — `/progress`

## 2. 最適合的資料狀態

| 項目 | 建議 |
|------|------|
| 體重曲線 | **4–8 週**緩降趨勢，平滑，**不要** cliff 暴跌 |
| 數字 | 體重可見，但**不要**把「-5.2 kg」做成海報大字 |
| 體脂 | 若有，變化 ≤ 2%，可信 |
| 達成率 | 若有，60–75% 區間 — **不要** 100% 假完美 |
| 週回饋 | 可見「下週已調整」類暗示，非 raw chart only |
| 色調 | 線條 sage / mocha，**不要** traffic light 紅綠燈 |

**情緒：** 安靜地變好 — Oura Readiness 曲線感，不是 MyFitnessPal 報表。

## 3. 標題文案

```
變化，慢慢發生。
```

## 4. 副標題

```
不用每天量。我們幫你看。
```

## 5. 是否需要真實食物照片

**否。** 純 progress UI。禁止 Before/After 人像。

| 避開 | 危險區域、刪除帳號、Apple Health |

**情緒：** 收束 — 像 Keynote 最後一張「One more thing」的 calm 版本。

## 4. 副標題

## 測試帳號準備（截圖前）

在正式站 `https://betterbit.app` 準備一個**示範帳號**，預先灌入：

1. 完成 onboarding（減脂目標、moderate activity）
2. 至少 **2 週** daily checkins + body measurements
3. Today 有 2–3 筆 food logs（含 1 筆 photo log）
4. 本週計畫含外食 + 運動
5. calorie bank 處於 **gentle recovery** 狀態（非 empty）
6. 訂閱狀態：**未訂閱**（Premium 頁可截）

```text
勿用：空白帳號、僅 1 天資料、全英文 UI、debug 文字
```
