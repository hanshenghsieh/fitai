# FitAI 市場分析：外食 vs 自己煮

## 📊 用戶市場分析

### 台灣健身用戶分布
```
自己煮健身族: 35-40%
├─ 健身愛好者 (重度用戶)
├─ 想省錢的上班族
└─ 時間充足的人

外食族: 50-55%
├─ 上班族 (時間壓力大)
├─ 不會煮的年輕人
└─ 忙碌的專業人士

混合型: 10-15%
└─ 根據時間在兩者之間切換
```

---

## 🍽️ 兩種方案的優缺點

### 方案 A: 只做自己煮 (現在的狀況)

**✅ 優勢:**
- 用戶體驗清晰 (輸入 = 執行)
- 數據精準 (每個食材份量明確)
- 開發簡單 (已完成)
- 成本低廉
- 用戶健康度最高
- 易於追蹤

**❌ 劣勢:**
- 市場只有 35-40%
- 排除上班族主力
- 對上班族說「自己煮」太不現實
- 會流失 50% 潛在用戶

**預期下載量:** 100K-500K

---

### 方案 B: 只做外食

**✅ 優勢:**
- 市場更大 (50-55%)
- 符合上班族需求
- 隨時隨地都能用

**❌ 劣勢:**
- 需要龐大餐廳數據庫
- 熱量數據來源複雜 (不準確)
- 用戶無法控制份量
- 健康效果減半
- 開發工作量 5 倍
- 需要外包合作

**預期下載量:** 200K-1M (但質量下降)

---

### 方案 C: 混合模式 (推薦) ⭐

**✅ 優勢:**
- 涵蓋 100% 用戶
- 用戶可自由選擇
- 高彈性
- 自己煮數據精準，外食也能用
- 逐步擴展，成本可控
- 貼近真實生活

**❌ 劣勢:**
- 功能更複雜
- 開發時間+2-3 週
- 需要餐廳數據源
- UI/UX 需要更多思考

**預期下載量:** 300K-2M (健康 + 方便)

---

## 🎯 台灣市場特殊考慮

### 台灣上班族痛點
```
早餐: 便利店買便當 (7-11, 全家)
午餐: 公司樓下食堂或外送
晚餐: 自己煮 或 外食
```

### 台灣優勢
- **便利店熱量資料完整** (7-11 有營養標示)
- **美食外送發達** (Uber Eats, foodpanda)
- **自己煮文化強** (媽媽族群)
- **健身房密集** (年輕人多)

---

## 📱 建議的實施策略

### 第一階段 (4-6 週) - 自己煮 MVP
```
✅ 推出自己煮版本 (你現在這裡)
✅ 上架 App Store / Play Store
✅ 收集用戶反饋
✅ 看自己煮族群是否夠活躍
```

### 第二階段 (6-8 週) - 添加外食功能
```
條件: 取決於第一階段反饋

如果自己煮用戶活躍度高 (>50% DAU):
  ➜ 不急著加外食
  ➜ 優先完善自己煮體驗
  ➜ 外食可能不是剛需

如果上班族強烈要求:
  ➜ 快速添加外食選項
  ➜ 從便利店菜單開始 (容易整合)
  ➜ 再擴展到餐廳
```

## 🔧 混合模式的具體實現

### UI 邏輯
```
今天的菜單選擇
├─ 選項 A: 自己煮 ✓ (詳細份量)
│  └─ 雞胸肉 160g → 1 塊 → 約手掌大
│
├─ 選項 B: 外食 
│  ├─ 便利店 (7-11, 全家)
│  │  └─ [便當選擇器]
│  │
│  ├─ 連鎖餐廳 (必勝客, 麥當勞)
│  │  └─ [菜單搜尋]
│  │
│  └─ 一般餐廳
│     └─ [手動輸入熱量]
│
└─ 選項 C: 混合 (早餐自己煮 + 午餐外食)
```

### 數據源整合
```
自己煮:
└─ 使用你的 21 種食物 ✅

外食:
├─ 便利店: 官方營養資訊 API
├─ 連鎖餐廳: Google Maps + 官方菜單
├─ 一般餐廳: 用戶輸入 (眾包)
└─ 外送: Uber Eats API

熱量估算:
└─ AI 識別 (拍照上傳菜色)
```

## 📈 市場需求調查

### 你應該先問新用戶

1. **你通常怎麼吃？**
   - 完全自己煮
   - 混合 (有時外食)
   - 通常外食

2. **外食時最困擾的是什麼？**
   - 不知道熱量
   - 怕吃太多
   - 選擇太多
   - 想堅持計畫但沒辦法

3. **理想的解決方案？**
   - 想要完全自己煮版本
   - 想要能追蹤外食的版本
   - 越靈活越好

## 🎯 我的最終建議

### 最聰明的做法: **混合模式 + 漸進式**

**第 1 版 (MVP - 4-6 週):**
```
✅ 完整的自己煮功能
✅ 簡單的「今天外食」記錄
   (只需輸入估計熱量，不需完整菜單)
✅ 上架測試市場反應
```

**第 2 版 (2-3 月後):**
```
根據用戶反饋:
├─ 如果外食需求高 → 加入便利店菜單
├─ 如果自己煮就夠用 → 專注優化深度功能
└─ 大多數是混合型 → 投入混合模式
```

**為什麼這樣做:**
1. **風險最低** - 先驗證市場
2. **成本最低** - 分階段投資
3. **最靈活** - 根據數據決策
4. **用戶滿意度最高** - 不強迫用戶
